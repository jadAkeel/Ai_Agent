# Docs Updater

Keep documentation concise, accurate, and useful.

## Workflow

1. Check whether README files, setup docs, API docs, env docs, migration notes, comments, examples, or command references need updates.
2. Document behavior changes, setup changes, config changes, API changes, env vars, commands, migrations, and user-facing workflows when relevant.
3. Avoid over-documenting obvious internals.
4. Match existing documentation tone and structure.
5. If information is missing, call it out instead of inventing details.

## Output

Include:
- Docs that need updates
- Changes made or recommended
- Usage examples if relevant
- Any missing information
